# 项目名称：Create: Kinetic Link (机械动力：动能链路)

## 1. 项目概述
本项目是一个 Minecraft 1.20.1 (Forge) 的 **Create (机械动力)** 附属 Mod。其核心功能是将游戏内的“应力 (Stress)”数据实时同步至 **DG-Lab (郊狼)** 郊狼基站/App。通过物理应力与体感强度的绑定，为玩家提供沉浸式的动力网络管理反馈。

## 2. 核心功能
*   **应力感应方块 (Stress Sensor Block)**：一种可接入传动轴的方块，能感知所在应力网络的实时负荷。
*   **动态强度映射**：
    *   **应力充沛 (Low Usage)**：当网络剩余应力充足时，输出较低强度的电流反馈。
    *   **负载上升 (High Usage)**：随应力百分比增加，线性提升电流强度。
    *   **网络过载 (Overloaded)**：当动力网崩溃时，立即触发最大预设强度的电流报警。
*   **双向通信**：集成 WebSocket 客户端，实现游戏与 DG-Lab App 的远程连接。

## 3. 技术架构与实现方式

### A. 机械动力集成 (Create API)
*   **方块类型**：继承 `KineticBlockEntity` 或 `SmartBlockEntity`，通过 `RotationPropagator` 接入动能网络。
*   **数据采集**：在 `tick()` 周期内，利用 `StressNetworkHandler` 获取当前网络的 `Capacity` (总容量) 与 `Usage` (已用量)。
*   **计算模型**：
    *   $StressRatio = Usage / Capacity$
    *   $Intensity = Base + (StressRatio \times Multiplier)$
    *   *异常处理：若 Capacity 为 0 或状态为 Overloaded，直接触发 Overload 逻辑。*

### B. 郊狼对接 (DG-Lab Protocol)
*   **通信协议**：基于 WebSocket 的 JSON 指令传输。
*   **参考逻辑**：参考 `DGLab-Craft` 项目，建立 `CoyoteSocketClient` 维护连接心跳。
*   **指令下发**：将计算出的强度值（0-100）转换为 DG-Lab 标准指令（如 `type: msg, data: strength-x-y`）。

### C. 开发环境
*   **Minecraft Version**: 1.20.1
*   **Loader**: Forge 47.2.0+
*   **Language**: Java 17
*   **Build Tool**: Gradle 8.x (使用 NeoForge/Forge Gradle 插件)
*   **Dependencies**: 
    *   `com.simibubi.create:create-1.20.1` (Create API)
    *   `org.java-websocket:Java-WebSocket` (用于 Socket 通信)

## 4. 关键代码逻辑参考
1.  **应力抓取**：通过 `getOrCreateNetwork()` 获取网络 ID，从服务端 Handler 提取数据。
2.  **数据平滑**：引入平滑滤波器（Smoothing Filter），防止应力小幅波动导致电流频繁震荡。
3.  **连接配置**：在游戏中提供配置文件（Config），允许用户输入 Socket 服务器 IP、端口及强度上限。

## 5. 后续开发重点
1.  **GUI 界面**：添加方块配置界面，用于绑定特定的 A/B 通道。
2.  **安全限制**：实现“安全熔断”机制，当心跳丢失或游戏卡顿时自动停止输出。
3.  **波形切换**：针对不同应力百分比，切换不同的脉冲波形（Pulse Patterns）。
